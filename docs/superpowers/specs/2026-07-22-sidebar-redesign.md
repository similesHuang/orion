# Orion 桌面端侧边栏与输入框 UI 精简设计

> 版本：v0.1
> 日期：2026-07-22
> 状态：草稿

---

## 1. 背景与问题

当前桌面端左侧边栏和输入框区域负载过重：

**侧边栏：**
- 品牌 Logo + 名称
- 3 个工具栏按钮（新建会话 / 添加 Project / 设置）
- Projects 折叠列表，每个 Project 下有重命名/刷新/删除/新建会话等操作按钮
- 独立会话列表
- 底部当前会话名 + 重命名/删除/导出/导入 4 个按钮

**输入框区域：**
- 文件附件按钮
- 斜杠命令提示
- Project 绑定条（绑定信息 + 解除绑定按钮）
- 模型选择器下拉框
- 拖拽上传覆盖层

过多的 UI 元素导致界面杂乱，使用效率低。

## 2. 设计目标

1. **精简侧边栏** — 只保留核心导航功能，去掉冗余操作入口
2. **简化输入框** — 只保留文本输入和发送
3. **重构设置入口** — 齿轮按钮 → 弹出菜单 → 分页设置面板
4. **视觉上更清爽** — 减少干扰元素，信息密度适中

## 3. 方案 B: Project 折叠 + 会话两级

### 3.1 侧边栏结构（自上而下）

```
┌─────────────────────────┐
│ Orion    logo       [+] │  ← 品牌 + 新建会话按钮
├─────────────────────────┤
│ ▼ my-project    main    │  ← Project 组（可折叠），显示 git 分支
│   ● 帮我整理项目文档     │  ← 选中态：青绿左边框 + 浅色背景
│     2小时前 · 12条消息   │
│     分析日志文件         │
│     昨天                │
│     部署配置讨论         │
│     3天前               │
│                         │
│ ▼ 独立会话           2  │  ← 独立会话组（可折叠），右侧数字
│     关于新功能的想法     │
│     1小时前             │
├─────────────────────────┤
│ [⚙ 设置]               │  ← 长方形按钮，hover 反馈
└─────────────────────────┘
```

### 3.2 各区域详情

**品牌区：** 左侧 Orion 文字 Logo，右侧「+」按钮新建会话，无其他工具栏按钮。

**Project 分组：**
- 可折叠 header，显示 Project 名称 + git 分支标签（蓝色徽标）
- 缩进展示子会话
- 每个会话条目：标题（单行省略）+ 时间说明（如"2小时前"或"昨天"）
- 当前会话：左侧 2px 青绿竖线 + 浅色背景
- 不再有 Project 级别的操作按钮（重命名/刷新/删除/新建会话）

**独立会话分组：**
- 可折叠 header，右侧显示会话计数
- 同 Project 下会话相同的条目样式

**底部设置按钮：**
- 长方形按钮，左侧齿轮图标 + "设置"文字
- hover 时背景色加深
- click 弹出 popover 菜单

### 3.3 设置菜单

点击底部设置按钮后弹出 popover（居中显示在按钮上方）：

```
┌──────────────────────┐
│ 🤖 模型配置           │  ← 第一个菜单项
│ 🔌 Gateway 配置    ✓  │  ← 右侧显示已配置状态
│ 📊 运行诊断           │
└──────────────────────┘
```

- 菜单项包含图标 + 文字描述
- 点击菜单项 → 关闭 popover → 打开对应的设置面板（复用现有 Drawer）
- Gateway 已配置时显示绿色 ✓ 状态标签

### 3.4 输入框

```
┌──────────────────────────────────────┐
│ 输入任务或问题...              [发送] │
└──────────────────────────────────────┘
```

- 纯文本输入框，glassmorphic 毛玻璃背景
- 右侧渐变「发送」按钮
- 无文件附件按钮
- 无模型选择器
- 无 Project 绑定条
- 无拖拽上传覆盖层
- 斜杠命令提示保留（输入 / 时浮出）

### 3.5 状态栏

从原来位置调整到对话区域顶部，更紧凑：

```
● Claude Opus 4          ↑1.2K ↓3.4K ⚡78%
```

- 左侧：Agent 状态指示点 + 模型名称
- 右侧：Token 消耗信息

## 4. 影响范围

### 新增组件
- `SettingsMenu` — 底部设置 popover 菜单（可复用 Ant Design Popover）

### 修改组件
- `App.tsx` — 删除侧边栏工具栏按钮、Project 操作按钮、底部操作按钮；替换为新的布局结构
- `App.tsx` — 输入框区域去掉附件按钮、模型选择器、Project 绑定条
- `App.tsx` — 设置触发方式从 Drawer 直接打开改为 popover 菜单 → Drawer

### 删除的 UI 元素
| 元素 | 位置 | 原因 |
|------|------|------|
| 添加 Project 按钮 | 侧边栏工具栏 | 用户可从文件菜单操作 |
| 模型选择器 | 输入框下方 | 设置面板中可切换 |
| 文件附件按钮 | 输入框 | 减少输入区域复杂度 |
| 拖拽上传覆盖层 | 输入框 | 同上 |
| Project 绑定条 | 输入框下方 | 在侧边栏中已可见 |
| Project 操作按钮（4个） | 侧边栏 Project | 减少干扰 |
| 底部当前会话操作（4个） | 侧边栏底部 | 减少干扰 |

### 保留但可能调整
- 对话导入/导出 — 可考虑移至设置菜单中
- 模型切换 — 在模型配置面板中操作
